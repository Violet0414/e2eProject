# 导出Excel

将结构化测试用例导出为 .xlsx 文件，符合公司模板格式。

## 输入

- 结构化测试用例（通常由"用例生成"技能产出）
- 可选：公司模板文件路径（默认使用 `./referenceDocument/用例模板.xlsx`，如用户指定则优先使用用户指定的路径）

## 处理规则

1. **确认Python环境**：检查是否安装了 `openpyxl` 库，未安装则先安装
2. **确定输出路径**：默认输出到 `./output/{当前日期}/{时间戳}_测试用例.xlsx`
3. **模板格式**（如无公司模板，使用以下默认格式）：

### 默认表头

| 列 | 表头名 | 说明 |
|----|--------|------|
| A  | 用例ID | TC-XXX-NNN |
| B  | 用例标题 | 不超过30字 |
| C  | 关联测试点 | TP编号 |
| D  | 所属模块 | 功能模块名 |
| E  | 优先级 | P0/P1/P2 |
| F  | 前置条件 | 执行前需满足的条件 |
| G  | 步骤编号 | 步骤序号 |
| H  | 操作步骤 | 具体操作描述 |
| I  | 测试数据 | 该步骤的输入数据 |
| J  | 预期结果 | 期望的输出/状态 |

### 格式要求

- 表头行：加粗、居中、浅蓝底色
- 列宽自适应内容
- 用例之间用细边框分隔
- 同一用例多个步骤时，ID/标题/前置条件等合并单元格
- Sheet名称：`测试用例`

4. **如提供公司模板**：读取模板文件（默认 `./referenceDocument/用例模板.xlsx`），按模板中的列顺序和格式输出，保留模板的样式

## 执行步骤

1. 解析输入的测试用例数据
2. 如用户提供模板路径，读取模板获取格式
3. 生成 .xlsx 文件到指定路径
4. 输出文件路径给用户确认

## 输出

- 文件路径：`./output/{当前日期}/testcases_{日期}.xlsx`
- 统计信息：用例总数、模块分布

## 注意事项

- 文件已存在时询问用户是否覆盖
- 导出完成后提示用户文件位置
- 如测试用例数据格式不符预期，提示用户先通过"用例生成"技能规范化
