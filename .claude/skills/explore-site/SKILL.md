# 页面探索

使用Playwright MCP对用户指定的页面URL进行功能模块完整探索，生成结构化探索记录，供后续 generate-testcases-from-explore 技能生成测试用例。

**本技能聚焦于用户指定的单个页面。操作后跳转的页面（如新增/编辑/详情表单页）通过子会话探索，防止主会话上下文溢出。**

## 文件路径规范

**所有运行时生成的文件统一放在 `~/e2eProject/exploreOutput/{当前日期}_{时间}/` 下，严禁在skills目录下创建任何文件或目录。**

- 探索记录：`~/e2eProject/exploreOutput/{当前日期}_{时间}/explore_record.md`
- 截图目录：`~/e2eProject/exploreOutput/{当前日期}_{时间}/screenshots/`
- 参考文档：`~/e2eProject/referenceDocument/testPointsRequirement.md`

## 输入

- **页面URL**：用户提供的待探索页面URL
- **登录凭据**（可选）：如页面需要登录，用户提供账号密码
- **参考文档**：`~/e2eProject/referenceDocument/testPointsRequirement.md`

## 探索流程

### 1. 初始化

- 创建 `~/e2eProject/exploreOutput/{当前日期}_{时间}/` 和 `~/e2eProject/exploreOutput/{当前日期}_{时间}/screenshots/` 目录
- 使用Playwright MCP打开浏览器，导航到目标页面URL
- 如页面需要登录，先执行登录操作再导航到目标页面
- 记录页面标题和当前URL

### 2. 主页面探索

聚焦当前页面，按以下类别进行探索。遇到跳转页面时，开启子会话探索（见第3步）。

#### 一、列表查询

- 记录查询条件字段及其类型（文本/日期/时间/树形/省市区级联等）
- 记录查询条件的默认提示文字
- 测试选择类查询条件的下拉选项
- 测试日期/时间类查询条件的控件
- 测试树形结构的展开收起、层级选项
- 测试省市区级联的过滤逻辑
- 测试单个查询条件查询、多条件组合查询、查询结果为空
- 测试翻页后查询、重置查询条件、展开收起查询条件
- 记录按钮名称

#### 二、列表数据

- 记录表头字段
- 记录字段值（新增后对比新增值、编辑后对比编辑值、无操作对比接口响应）
- 查询后列表数据、删除后列表数据

#### 三、列表翻页

- 记录翻页控件样式
- 测试页码点击、页码输入、上一页/下一页、首页/末页
- 测试每页条数切换、翻页后数据显示

#### 四、列表操作

- 记录操作按钮名称
- 记录操作按钮与数据状态的对应关系
- 测试删除操作（确认弹窗、删除后列表不显示）
- 测试批量删除（确认弹窗、未选择提示、查询后全选）
- 测试其他批量操作（未选择提示、选择后操作所选数据）

#### 五、接口监测

- 记录探索过程中观察到的接口调用（URL、请求方法、请求参数、响应格式）

### 3. 跳转子会话探索

当主页面操作导致页面跳转时，必须使用Agent工具开启子会话进行探索，防止主会话上下文溢出。

**触发条件**：
- 点击新增按钮 → 新增表单页
- 点击编辑按钮 → 编辑表单页
- 点击详情按钮 → 详情表单页
- 其他导致页面跳转的操作

**子会话执行方式**：
使用Agent工具spawn子Agent，子Agent接收跳转页面的上下文信息，独立完成探索后将结构化结果返回给主会话。

**子会话输入**：
- 跳转触发操作（如"点击新增按钮"）
- 跳转后页面URL
- 页面类型（新增表单/编辑表单/详情表单/其他）
- 参考文档路径：`~/e2eProject/referenceDocument/testPointsRequirement.md`

**子会话探索内容**：

**新增表单页**：
- 页面标题
- 所有字段及其类型、默认提示、必填标识
- 逐字段按类型测试并记录（单文本/多行文本/下拉选择/多选/日期/时间/金额/身份证/手机号/地图选址/附件/图片）
- 必填项验证（不填提示、填写保存）
- 非必填项验证（不填保存、填写保存）
- 按钮操作（关闭/保存提交/取消）
- 保存后列表显示该数据、字段值正确

**编辑表单页**：
- 字段回填验证（与新增值对比）
- 清空非必填保存
- 关闭不更新、保存后更新、取消不更新
- 附件回显

**详情表单页**：
- 字段名称和值（与新增/编辑一致）
- 图片预览、附件下载
- 子TAB切换和内容
- 返回/关闭按钮
- 空字段值显示、长字段值显示效果

**子会话返回结果**：
子Agent完成后，将探索结果以结构化文本返回，主会话将结果整合到探索记录的对应章节中。

### 4. 记录规范

只记录 generate-testcases-from-explore 技能生成测试用例所需的必要信息：
- 元素的字段名、类型、默认提示、选项值
- 校验规则和测试结果
- 操作触发条件和结果
- 接口调用信息

## 持久化

- 探索记录写入 `~/e2eProject/exploreOutput/{当前日期}_{时间}/explore_record.md`
- 截图保存到 `~/e2eProject/exploreOutput/{当前日期}_{时间}/screenshots/`（按需截图，文件名格式：`{序号}_{描述}.png`）

## 注意事项

- 只探索用户指定的页面及其操作跳转的页面，不主动遍历其他菜单或页面
- 对当前页面及跳转页面的功能模块探索必须全面完整，覆盖testPointsRequirement.md中适用的测试点
- 操作跳转页面时必须使用子会话（Agent工具）探索，避免主会话上下文溢出
- 探索记录要结构化、详细，确保后续可直接用于生成测试用例
- **必须记录 route_path**：从页面URL中提取路径部分（如 `/business/publicFacilities/chargingPile`），写入探索记录头部，供后续 test-script-generate 技能生成带跳转路径的测试用例
- 如遇错误或无法完成的操作，记录原因并继续探索其他部分
- **严禁在skills目录下创建任何文件、目录或安装依赖**
- 探索完成后清理Playwright MCP产生的临时文件
